# Proyecto Churn MLOps: Predicción y Explicabilidad

Se documenta todo el ciclo de vida del análisis, modelado e interpretabilidad de un modelo de Machine Learning diseñado para predecir la fuga de clientes (Customer Churn) en el sector de las telecomunicaciones.

## Objetivo del Proyecto

El objetivo principal de este proyecto es desarrollar un modelo predictivo robusto que identifique qué clientes tienen mayor probabilidad de abandonar el servicio. Además, buscamos integrar herramientas de **MLOps** y explicar las decisiones del modelo para generar confianza técnica y accionabilidad para el negocio.

##  Indice

El proceso está documentado paso a paso a través de notebooks de Jupyter, abarcando las siguientes etapas clave:

1. **Análisis Exploratorio de Datos (EDA)**: Exploración inicial del dataset, análisis de comportamiento de las variables, detección de patrones correlacionados al churn y preparación de los datos.
2. **Entrenamiento y Evaluación de Modelos**: Proceso de modelado con manejo avanzado de clases desbalanceadas (SMOTE), entrenamiento de modelos (como CatBoost), búsqueda de hiperparámetros y optimización de umbrales.
3. **Explicabilidad con LIME**: Uso de *Local Interpretable Model-agnostic Explanations (LIME)* para abrir la "caja negra" del modelo, entendiendo a nivel individual qué factores impulsan la predicción de cancelación de un cliente.
